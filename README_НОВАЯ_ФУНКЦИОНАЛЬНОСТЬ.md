# Новая функциональность: Команда "Журнал регистрации по объекту"

## Обзор изменений

В расширение "Логгирование изменений данных" добавлена новая функциональность для быстрого открытия журнала регистрации с отбором по текущему объекту.

## Что добавлено

### 1. Расширен общий модуль `ЛогИ_ОбщийМодуль`

Добавлены новые экспортные процедуры:

```bsl
Процедура ОткрытьЖурналРегистрацииПоОбъекту(ТекущийОбъект) Экспорт
Процедура КомандаОткрытьЖурналРегистрации(Форма) Экспорт
```

### 2. Создан новый общий модуль `ЛогИ_ОбщийМодульВызовСервера`

Модуль для серверных вызовов с функциями:

```bsl
Процедура КомандаОткрытьЖурналРегистрации(Форма) Экспорт
Функция ПолучитьДанныеДляОтбора(ТекущийОбъект) Экспорт
```

### 3. Создана общая команда `ЛогИ_ОткрытьЖурналРегистрации`

- **Название**: "Журнал регистрации по объекту"
- **Горячая клавиша**: Ctrl+Shift+L
- **Иконка**: StdPicture.EventLogMonitor
- **Доступна во всех формах объектов**

### 4. Создана обработка `ЛогИ_ДобавитьКомандуВФормы`

Служебная обработка для автоматического добавления команды в существующие формы.

## Как использовать

### Автоматическое использование

Общая команда `ЛогИ_ОткрытьЖурналРегистрации` автоматически доступна во всех формах объектов. Для её использования:

1. Откройте любую форму документа, справочника или константы
2. Нажмите **Ctrl+Shift+L** или найдите команду "Журнал регистрации по объекту" в интерфейсе
3. Откроется журнал регистрации с отбором по текущему объекту

### Ручное добавление в формы

Если общая команда не отображается автоматически, добавьте её вручную:

1. Откройте форму в конфигураторе
2. Добавьте команду с действием `ЛогИ_ОткрытьЖурналРегистрации`
3. Добавьте кнопку команды в интерфейс формы

## Поддерживаемые типы объектов

Команда работает с объектами, которые логируются системой:

- ✅ **Документы** (все виды)
- ✅ **Справочники** (все виды)
- ✅ **Планы видов расчета**
- ✅ **Константы**
- ✅ **Регистры сведений** (записи)
- ✅ **Регистры накопления** (записи)

## Принцип работы

1. Команда получает текущий объект из формы
2. Определяет правильную ссылку на объект
3. Формирует отбор для журнала регистрации:
   - **Событие**: "Лог. Изменение данных"
   - **Данные**: текущий объект
4. Открывает журнал регистрации с применённым отбором

## Преимущества

- 🚀 **Быстрый доступ** к истории изменений объекта
- 🎯 **Точный отбор** только по нужному объекту
- ⌨️ **Горячая клавиша** для быстрого вызова
- 🔄 **Универсальность** - работает во всех формах
- 📋 **Стандартная интеграция** с платформой 1С

## Ограничения

- Команда работает только для сохранённых объектов
- Требует наличие записей в журнале регистрации для данного объекта
- Отбор применяется только по событию "Лог. Изменение данных"

## Технические детали

### Структура отбора

```bsl
Отбор = Новый Структура;
Отбор.Вставить("Событие", "Лог. Изменение данных");
Отбор.Вставить("Данные", ТекущийОбъект);
```

### Получение объекта из формы

Функция автоматически определяет тип формы и получает объект:

- Для форм объектов: `Форма.Объект.Ссылка` или `Форма.Объект`
- Для форм записей: `Форма.Запись`
- Для форм констант: `Форма.Константа`

- ✅ Совместимо с 1С:Предприятие 8.3.11+
- ✅ Работает в управляемом приложении
- ✅ Поддерживает веб-клиент и тонкий клиент
- ✅ Не конфликтует с существующей функциональностью

## Примеры использования

### Анализ изменений документа

1. Откройте документ (например, "Поступление товаров")
2. Нажмите Ctrl+Shift+L
3. Увидите все изменения этого конкретного документа

### Отслеживание изменений справочника

1. Откройте элемент справочника (например, "Номенклатура")
2. Используйте команду "Журнал регистрации по объекту"
3. Просмотрите историю всех изменений элемента

### Контроль изменений констант

1. Откройте форму константы
2. Вызовите команду для просмотра истории изменения значения
